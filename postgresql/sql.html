5.2 DML

Data Manipulation Language - used for adding (inserting), deleting,
selecting, and modifying (updating) data in a database.

5.2.1 Insert

INSERT — create new rows in a table

create table sample (id int);

insert into sample (id)
values (1) ,(2) ,(3);

select * from sample;

insert into sample(id)
select * from generate_series(100,105);

truncate sample;

⚡ VALUES — compute a set of rows

values (1) ,(2) ,(3);

values (now()), (current_timestamp);

select now();

5.2.2 Select

SELECT — retrieve rows from a table or view
SELECT INTO — define a new table from the results of a query

select * from sample;

select * into sample_big from sample where id > 102;

select * from sample_big;

5.2.3 Update

UPDATE — update rows of a table

update sample
set id = (id + 100)
where id > 100
returning id

select * from sample

5.2.4 Delete

DELETE — delete rows of a table

delete from sample
where id >= 100
returning id;

Delete VS Truncate

⚡ TRUNCATE — empty a table or set of tables

5.3 DCL

Data Control Language - defines the control over the data in the database.

5.3.1 Grant

GRANT — define access privileges

create user samurai;

\du

GRANT ALL PRIVILEGES ON sample TO samurai;

SELECT grantee, privilege_type
FROM information_schema.role_table_grants
WHERE table_name='sample'

5.3.2 Revoke

REVOKE — remove access privileges

REVOKE ALL PRIVILEGES ON sample FROM samurai;

SELECT grantee, privilege_type
FROM information_schema.role_table_grants
WHERE table_name='sample'

5.4 TCL

Transaction Control Language - commands are used to manage the transactions in the database.
Samples in ./playground.sql file.

  • Begin
    BEGIN — start a transaction block
    START TRANSACTION — start a transaction block
    PREPARE TRANSACTION — prepare the current transaction for two-phase commit
  • Commit
    COMMIT — commit the current transaction
    COMMIT PREPARED — commit a transaction that was earlier prepared for two-phase commit
    END — commit the current transaction
  • Rollback
    ROLLBACK — abort the current transaction
    ROLLBACK PREPARED — cancel a transaction that was earlier prepared for two-phase commit
    ROLLBACK TO SAVEPOINT — roll back to a savepoint
  • Abort
    ABORT — abort the current transaction

  • Savepoint
    SAVEPOINT — define a new savepoint within the current transaction
    RELEASE SAVEPOINT — destroy a previously defined savepoint

    begin;
insert into sample (id)
select * from generate_series(0, 10);
commit;

    begin;
delete from sample;
select * from sample;
rollback;

Operator Argument Result
-> integer, text jsonb
->> integer, text text
#> text[] jsonb
#>> text[] text

    x -> 1 as "->"
  , pg_typeof(x -> 1) as "->"
  , x->> 1 as "->>"
  , x#> '{0, name}' as "->>"

8.1 Sorting and Filtering

select array_agg(x)
from  generate_series(0, 10) x

select array_agg(x::int order by x desc)
from  generate_series(0, 10) x

select array_agg(x::int order by x desc) filter (where x > 5)
from  generate_series(0, 10) x

select unnest(x)
from (
  select array_agg(x::int order by x desc) filter (where x > 5)
  from  generate_series(0, 10) x
) t(x)

8.2 Window Functions

Window Functions - performs a calculation across a set of table rows that are somehow related to the current row.

select EXTRACT(YEAR from d), v
      , lag(v, -1) over (order by d desc) prev
      , v - lag(v, -1) over (order by d desc)  inc
from (values ('2021-01-01'::timestamp, 10)
	   , ('2020-01-01'::timestamp, 8)
	   , ('2019-01-01'::timestamp, 10)
	   , ('2018-01-01'::timestamp, 5))  t(d, v)

10.1 Recursive

WITH RECURSIVE r AS (
    SELECT
	1 AS i,
	1 AS factorial

    UNION

    SELECT
	i+1 AS i,
	factorial * (i+1) as factorial
    FROM r
    WHERE i < 10
)
SELECT * FROM r;
