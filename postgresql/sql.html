    CREATE TABLE — define a new table

    How to read Synopsis:

    • [ a | b ] means that a or b is optional
    • { a | b } means a or b
    • Bold text represents something you need to fill.
    • … - mean that the preceding element can be repeated
    • , … - mean that the preceding element can be repeated with separator ,

    CREATE [ [ GLOBAL | LOCAL ] { TEMPORARY | TEMP } | UNLOGGED ] TABLE [ IF NOT EXISTS ] tablename ([{ columnname datatype [ COLLATE collation ] [ columnconstraint [ ... ] ] | tableconstraint | LIKE sourcetable [ likeoption … ] } [, … ] ]) [ INHERITS ( parenttable [, … ] ) ] [ ~PARTITION BY~ ={ RANGE | LIST | HASH } ( { columnname | ( expression ) } [ COLLATE collation ] [ opclass ] [, … ] ) ] [ USING method ] [ WITH ( *storageparameter [ value ] [, … ] ) | WITHOUT OIDS ] [ ON COMMIT { PRESERVE ROWS | DELETE ROWS | DROP } ] [ TABLESPACE tablespacename ]

5.2 DML

Data Manipulation Language - used for adding (inserting), deleting, selecting, and modifying (updating) data in a database.

5.2.1 Insert

INSERT — create new rows in a table 

create table sample (id int);

insert into sample (id)
values (1) ,(2) ,(3);

select * from sample;

insert into sample(id)
select * from generate_series(100,105);

truncate sample;

⚡ VALUES — compute a set of rows 

values (1) ,(2) ,(3);

values (now()), (current_timestamp);

select now();

5.2.2 Select

SELECT — retrieve rows from a table or view SELECT INTO — define a new table from the results of a query 

select * from sample;

select * into sample_big from sample where id > 102;

select * from sample_big;

5.2.3 Update

UPDATE — update rows of a table 

update sample
set id = (id + 100)
where id > 100
returning id

select * from sample

5.2.4 Delete

DELETE — delete rows of a table 

delete from sample
where id >= 100
returning id;

Delete VS Truncate

⚡ TRUNCATE — empty a table or set of tables

5.3 DCL

Data Control Language - defines the control over the data in the database.

5.3.1 Grant

GRANT — define access privileges 

create user samurai;

\du

GRANT ALL PRIVILEGES ON sample TO samurai;

SELECT grantee, privilege_type
FROM information_schema.role_table_grants
WHERE table_name='sample'

5.3.2 Revoke

REVOKE — remove access privileges 

REVOKE ALL PRIVILEGES ON sample FROM samurai;

SELECT grantee, privilege_type
FROM information_schema.role_table_grants
WHERE table_name='sample'

5.4 TCL

Transaction Control Language - commands are used to manage the transactions in the database. Samples in ./playground.sql file.

  • Begin BEGIN — start a transaction block START TRANSACTION — start a transaction block PREPARE TRANSACTION — prepare the current transaction for two-phase commit
  • Commit COMMIT — commit the current transaction COMMIT PREPARED — commit a transaction that was earlier prepared for two-phase commit END — commit the current transaction
  • Rollback ROLLBACK — abort the current transaction ROLLBACK PREPARED — cancel a transaction that was earlier prepared for two-phase commit ROLLBACK TO SAVEPOINT — roll back to a savepoint
  • Abort ABORT — abort the current transaction

  • Savepoint SAVEPOINT — define a new savepoint within the current transaction RELEASE SAVEPOINT — destroy a previously defined savepoint 

    begin;
insert into sample (id)
select * from generate_series(0, 10);
commit;

    begin;
delete from sample;
select * from sample;
rollback;

5.5 PGSQL

CALL — invoke a procedure CHECKPOINT — force a write-ahead log checkpoint CLOSE — close a cursor COMMENT — define or change the comment of an object DECLARE — define a cursor DISCARD — discard session state DO — execute an anonymous code block FETCH — retrieve rows from a query using a cursor IMPORT FOREIGN SCHEMA — import table definitions from a foreign server LISTEN — listen for a notification LOAD — load a shared library file LOCK — lock a table MOVE — position a cursor NOTIFY — generate a notification REASSIGN OWNED — change the ownership of database objects owned by a database role REFRESH MATERIALIZED VIEW — replace the contents of a materialized view SECURITY LABEL — define or change a security label applied to an object UNLISTEN — stop listening for a notification VALUES — compute a set of rows

CLUSTER — cluster a table according to an index REINDEX — rebuild indexes

TRUNCATE — empty a table or set of tables COPY — copy data between a file and a table

Optimization & Analyze ANALYZE — collect statistics about a database EXPLAIN — show the execution plan of a statement VACUUM — garbage-collect and optionally analyze a database

Statement PREPARE — prepare a statement for execution EXECUTE — execute a prepared statement DEALLOCATE — deallocate a prepared statement

Run-time parameter SET — change a run-time parameter SET CONSTRAINTS — set constraint check timing for the current transaction SET ROLE — set the current user identifier of the current session SET SESSION AUTHORIZATION — set the session user identifier and the current user identifier of the current session SET TRANSACTION — set the characteristics of the current transaction SHOW — show the value of a run-time parameter RESET — restore the value of a run-time parameter to the default value

6.1 Numeric Types

Numeric Types - Numeric types consist of two-, four-, and eight-byte integers, four- and eight-byte floating-point numbers, and selectable-precision decimals.

Name StorageSize Description Range
smallint, int2 2 bytes small-range integer -32768 to +32767
integer, int, int4 4 bytes typical choice for integer -2147483648 to +2147483647
bigint, int8 8 bytes large-range integer -9223372036854775808 to +9223372036854775807
decimal, decimal [(p, s)] variable user-specified precision, exact up to 131072 digits before the decimal point; up to 16383 digits after the decimal point
numeric, numeric [(p, s)] variable user-specified precision, exact up to 131072 digits before the decimal point; up to 16383 digits after the decimal point
real, float4 4 bytes variable-precision, inexact 6 decimal digits precision
double precision, float8 8 bytes variable-precision, inexact 15 decimal digits precision
smallserial, serial2 2 bytes small autoincrementing integer 1 to 32767
serial, serial4 4 bytes autoincrementing integer 1 to 2147483647
bigserial, serial8 8 bytes large autoincrementing integer 1 to 9223372036854775807

6.6 Geometric Types

Geometric Types - Geometric data types represent two-dimensional spatial objects.

:zap:Geometric Functions and Operators

:zap:PostGIS - is a spatial database extender for PostgreSQL object-relational database. It adds support for geographic objects allowing location queries to be run in SQL.

Nam Storage Siz Description Representation
point 16 bytes Point on a plane (x,y)
line 32 bytes Infinite line {A,B,C}
lseg 32 bytes Finite line segment ((x1,y1),(x2,y2))
box 32 bytes Rectangular box ((x1,y1),(x2,y2))
path 16+16n bytes Closed path (similar to polygon) ((x1,y1),…)
path 16+16n bytes Open path [(x1,y1),…]
polygon 40+16n bytes Polygon (similar to closed path) ((x1,y1),…)
circle 24 bytes Circle <(x,y),r> (center point and radius) 
select POINT(10, 10);

select LSEG '(-10, 0), (10, 0)';

select LSEG '(-10, 0), (10, 0)' ?# LSEG '(0, -10), (0, 10)';

6.9 Other Types

Monetary - type stores a currency amount with a fixed fractional precision Binary Data Types - bytea data type allows storage of binary strings Enumerated Types - types are data types that comprise a static, ordered set of values Network Address - data types to store IPv4, IPv6, and MAC addresses Bit String - can be used to store or visualize bit masks. Text Search - data types that are designed to support full text search UUID type - for UUID storage XML type - data type can be used to store XML data Composite type - composite type represents the structure of a row or record Range types - are data types representing a range of values of some element type Pseudo-Types - special-purpose entries that are collectively called pseudo-types. A pseudo-type cannot be used as a column data type, but it can be used to declare a function's argument or result type.
